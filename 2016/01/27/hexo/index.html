hexo有哪些功能？

  • 让程序员码字
  • 让程序员有逼格的码字

如何开始？

分2个大步骤，这样清晰明了，以下仅限Windows，*OS X， Linux*下安装大同小易，比在Windows下更顺手

  • 第一步：本地搭建

    • 安装 git for Windows
    • 安装 Node
    • 在目标位置新建一个目录，用于存放blog的相关文件，例如我在D盘下新建了hexo目录，以下以D:\hexo目录作为示例

    • 打开D:\hexo目录，在此目录下右键Git Bash，在命令行窗口中执行以下命令：

      • npm install -g hexo
      • hexo init
      • 此时hexo就在当前目录（D:\hexo）下建立了相关文件
      • 继续执行 npm install 安装modules
      • 至此如果没有出现错误，大功告成，本地安装已完毕
      • 在命令行中继续执行 hexo server 回显如下即已启动本地hexo服务：

        $ hexo server
        [info] Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

      • 在浏览器中输入 http://localhost:4000/ 即可访问
      • 配置hexo：hexo配置文件分2个，一个是整站配置文件D:\hexo\_config.yml，另一个是主题配置文件，位于D:\hexo\themes\[相应主题目录]\_config.yml，具体的配置单项参考此文 hexo系列教程之配置文件，配置修改完成之后，可以重新运行hexo server -g后刷新浏览器查看一下效果

    • hexo 基本命令介绍

      hexo new "postName" #新建文章
      hexo new page "pageName" #新建页面
      hexo generate #生成静态页面至public目录
      hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
      hexo deploy #将.deploy目录部署到GitHub
      hexo deploy -g #复合命令，相当于hexo generate; hexo deploy;下同
      hexo server -g #复合命令，相当于hexo generate; hexo server;
      hexo n == hexo new #简写，下同
      hexo g == hexo generate
      hexo s == hexo server
      hexo d == hexo deploy

  • 第二步：部署到github
    • 添加本地ssh-key到github账户在下，如果之前做过可免，直接从下一条开始
      • 配置git

        git config –global user.email “xx@xx.com” [xxx替换自己的邮箱]
        git config –global user.name “xxx” [xxx替换自己想设置的名字，任意都可以]

      • 在git-bash中输入以下命令，一路回车，生成完成之后，会在用户的主目录（C:\Users\xxx\）下生成.ssh/目录，目录下有两个文件id_rsa，id_rsa.pub，用记事本打开id_rsa.pub，复制全部内容，在github账户管理下，SSH Keys，添加一个key，粘贴刚刚复制的内容，保存即可

        ssh-keygen -t rsa -C “[[你的邮箱]]”

    • 在github新建项目，项目名一定要设置成xxx.github.io，xxx是你的github账户名
    • 打开整站配置文件D:\hexo\_config.yml，找到deploy字段（一般在最后），进行如下配置：

      deploy:
      type: github
      repository: git@github.com/xxx/xxx.github.io.git
      branch: master

    • 运行 hexo deploy -g，即可成功将blog部署到github，通过外网，访问xxx.github.io，就可以看到效果了，新建文章之后，运行一次 hexo deploy -g，完成之后即可访问xxx.github.io看到更新
