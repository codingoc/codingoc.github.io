常见问题

  1. 如何免密码与远程仓库同步？
    git可以使用两种协议来登录服务器进行同步，一种是HTTP(S)，一种是ssh，下面以Github为例：
    在Github中可以看到下图的情形，如果你没有登录Github账号只能看到HTTPS

    如果你用HTTPS URL来clone，push项目，这时你就需要输入github的账户名和密码，这样不方便，下面介绍使用ssh免密码同步
    ssh免密码使用的技术是ssh认证登录，ssh认证登录更多参见blog中关于ssh的讲解，下面只说如何配置Github免密码同步

    Windows | OS X | Linux

    Git Bash中输入 ssh-keygen -t rsa ，rsa是指使用RSA加密算法，一路回车，提示输入的部分不用理会，之后你会在用户主目录[Windows：C:\Users\xxx]或者[OS X | Linux：~]下看到一个.ssh目录，里面有2个文件，id_rsa，id_rsa.pub，这两个都是文本文件，前者是RSA私钥，后者是RSA公钥，现在我们 只需要将公钥id_rsa.pub中的内容复制[复制时只复制 ssh-rsa到xxx@xxx前的内容，也就是说复制不把文件最后的xxx@xxx复制进去]到Github个人设置->SSH keys->Add SSH key，如下图，

    添加完成之后，可以在Git Bash中输入 ssh -T git@github.com 测试是否成功，如果输出以下内容则表明成功，以后与Github同步就不用不输入账号密码了，推荐所有的项目都用ssh协议
    $ ssh -T git@github.com
    Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access.

  2. 如何添加多个远程仓库，如何同步到多个远程仓库？
    git add remote add [remotename] [url] #使用此命令添加远程仓库，[remotename]为远程仓库名，随意，例如backup
    git remote -v #使用此命令查看远程仓库列表

  3. 如何配置全局gitignore文件？
    有时候需要在git自动忽略某些文件或者目录，例如IDE生成的build/目录，*.exe文件等，单独在项目下每次都要加入比较麻烦，我们可以建立一个全局gitignore文件，添加这些需要忽略的文件

    在用户主目录下，新建文件 *.gitignore_global*
    Windows用户主目录一般是”C:\Users\xxx”[xxx是用户名]，Windows下不能直接新建这个文件，在Git Bash中使用touch .gitignore_global来创建，OS X和Linux中直接cd到用户主目录，使用touch .gitignore_global创建，此文件就是文本文件，Windows下可用记事本打开，添加忽略项
    如果你在一个项目下需要将全局忽略的文件添加到git跟踪，例如你.gitignore_global的中有添加*.exe（忽略所有的exe文件），但在实际项目中你有个叫tools.exe的文件需要git进行跟踪怎么办？这时你只需要在该项目下的.gitignore文件中添加!tools.exe即可（前面添加一个感叹号，表示进行跟踪，不忽略）
